## 專案背景

我要做一個「00631L（元大台灣50正2）波段進出場訊號」的監控工具，純顯示用，**不做自動下單、不做即時推播提醒**，我自己開網頁進去看就好。

**策略邏輯（已確認）：**

- 空手狀態下，追蹤股價相對週線（MA5）、月線（MA20）的位置
- 跌破週線（MA5）→ 買進訊號，投入預定資金的 1/2
- 續跌破月線（MA20）→ 買進訊號，投入剩餘的 1/2（此時滿倉）
- 站回月線（MA20）之上 → 賣出訊號，全數出場，回到空手狀態，重新等待下一次訊號
- 不無限往下加碼，跌破月線後即使繼續下跌也不再加碼（這條先寫死，之後可能再討論要不要加安全閥停損）

**重要技術細節：**

- 00631L 在 2026/3/31 以 1 股拆 22 股的比例完成分割，分割前後價格不連續。所有歷史資料在做均線計算與回測時，**必須做股票分割還原（split-adjusted）**，否則分割當天會出現假跳空，訊號會失真。
- 資料來源可用台灣證交所 TWSE 開放資料或 FinMind，抓收盤價即可，不需要即時報價。

**架構決定：**

- 每個交易日收盤後，用排程（GitHub Actions cron）跑一次 Python 批次腳本，抓當日收盤價、算 MA5/MA20、判斷目前訊號狀態，寫成一個 JSON 檔並 commit 回 repo
- 前端是純靜態網頁，讀這個 JSON 顯示：目前股價、週線值、月線值、目前部位狀態、目前訊號
- 部署在 GitHub Pages，不用另外租主機
- GitHub Actions 排程時間用 UTC，台股收盤 13:30（台灣時間）建議排在台灣時間 14:00 左右（即 UTC 06:00），保留資料結算緩衝時間
- 腳本要自己判斷「今天是不是有新的交易日資料」，非交易日（假日、颱風假）要跳過，不要誤判

**目標：** 寫一支腳本，能抓取 00631L 從掛牌至今的歷史日收盤價，並正確處理 2026/3/31 的 1:22 分割還原，輸出成乾淨的時間序列（date, close）。

**要做的事：**

- 選定資料來源（TWSE OpenAPI 或 FinMind，跟我確認後再動工）
- 抓歷史收盤價
- 實作分割還原邏輯（分割日之前的價格全部除以 22，或用等效的還原方式）
- 輸出成 CSV 或 parquet，方便 P2 直接讀取
- 寫一個簡單的驗證：印出分割日前後幾天的還原後價格，確認銜接平順、沒有假跳空

**驗收方式：** 我會看分割日（2026/3/31）前後的還原股價，確認是連續平滑的，不是一根巨大跳空。

## P2｜策略邏輯與回測

**目標：** 把 P1 的歷史資料套上策略邏輯，跑歷史回測，看這套規則過去表現如何，再決定要不要拿去做成即時訊號。

**要做的事：**

- 計算 MA5（週線）、MA20（月線）
- 實作訊號狀態機：空手 → 跌破週線買半倉 → 跌破月線買滿倉 → 站回月線全出場 → 回到空手
- 回測時記錄每一次進出場的日期、價格、報酬
- 輸出統計：總進出場次數、平均持有天數、累積報酬率、最大回撤、勝率
- 用簡單的圖表畫出股價走勢 + 標出每次買賣點，方便肉眼檢查邏輯有沒有跑錯

**驗收方式：** 我會看幾筆手動抽查的進出場點是否符合邏輯，並看整體統計數字是否合理。

## P3｜每日訊號批次腳本

**目標：** 把 P2 的訊號邏輯，改寫成「只看最新狀態」的每日批次腳本，輸出給前端用的 JSON。

**要做的事：**

- 讀取最新收盤價（銜接 P1 的資料源）
- 判斷今天是否為新交易日，不是的話直接跳過、不更新
- 計算當前 MA5、MA20，判斷目前部位狀態與訊號
- 輸出 JSON，欄位大致包含：日期、收盤價、MA5、MA20、目前部位狀態（空手/半倉/滿倉）、目前訊號（無/買進-破週線/買進-破月線/賣出）
- 腳本要是冪等的（同一天重複跑不會產生錯誤資料）

**驗收方式：** 我手動跑一次，比對輸出的 JSON 數字跟我自己算的是否一致。

**要做的事：**
